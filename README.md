# MiniVM

A stack-based bytecode Virtual Machine written in Rust.

MiniVM is capable of assembling a custom assembly language into bytecode, executing the generated bytecode, and disassembling bytecode back into assembly. The project demonstrates the complete execution pipeline of a simple programming language backend.

---

# Features

* Stack-based Virtual Machine
* Custom Instruction Set Architecture (ISA)
* Assembler (`.tasm` → `.tbc`)
* Disassembler (`.tbc` → `.tasm`)
* Binary bytecode format
* Bytecode encoding and decoding
* Bytecode file validation using magic number and version
* Global memory slots (`LOAD` / `STORE`)
* Trace mode for debugging
* Command Line Interface (CLI)
* Comprehensive acceptance test programs

# Architecture

The execution pipeline is:

```text
Assembly Source (.tasm)

        │

        ▼

Assembler

        │

        ▼

Vec<Op>

        │

        ▼

Bytecode Encoder

        │

        ▼

Bytecode File (.tbc)

        │

        ▼

Bytecode Reader

        │

        ▼

Bytecode Decoder

        │

        ▼

Virtual Machine

        │

        ▼

Program Output
```

For disassembly:

```text
.tbc

↓

Bytecode Reader

↓

Decoder

↓

Vec<Op>

↓

Disassembler

↓

.tasm
```

# Instruction Set

## Stack Operations

| Instruction | Description               |
| ----------- | ------------------------- |
| PUSH x      | Push value onto the stack |
| POP         | Remove top element        |
| DUP         | Duplicate top element     |
| SWAP        | Swap top two elements     |

## Arithmetic Operations

| Instruction | Description      |
| ----------- | ---------------- |
| ADD         | Addition         |
| SUB         | Subtraction      |
| MUL         | Multiplication   |
| DIV         | Integer Division |
| MOD         | Modulus          |
| NEG         | Negation         |

## Memory Operations

| Instruction | Description            |
| ----------- | ---------------------- |
| LOAD n      | Load global slot       |
| STORE n     | Store into global slot |

## System Operations

| Instruction | Description        |
| ----------- | ------------------ |
| PRINT       | Print top of stack |
| HALT        | Stop execution     |

# Binary File Format

Every compiled bytecode file (`.tbc`) follows this layout:

```text
+--------------------+
| Magic Number       | 4 bytes
+--------------------+
| Version            | 1 byte
+--------------------+
| Bytecode Length    | 4 bytes
+--------------------+
| Program Bytecode   | N bytes
+--------------------+
```

Magic Number:

```
4D 56 4D 00
```

Version:

```
01
```

The VM validates the file signature and bytecode length before execution.

# CLI Usage

### Assemble

```bash
cargo run -- asm input.tasm output.tbc
```

### Run

```bash
cargo run -- run output.tbc
```

### Run with Trace

```bash
cargo run -- run output.tbc --trace
```

### Disassemble

```bash
cargo run -- dis output.tbc
```

# Trace Mode

Trace mode prints every instruction before execution together with the stack contents before and after execution.

Example:

```text
Instruction : Push(10)
Before Stack: []

After Stack : [10]

--------------------------
```

This feature is useful for debugging and understanding stack execution.

# Design Decisions

* Implemented as a **stack-based virtual machine** because stack architectures are simpler to design and require fewer instruction operands.
* Used **Little Endian** encoding to serialize integers into bytecode.
* Separated the project into modules (Assembler, VM, ISA, File I/O, Disassembler) to keep responsibilities independent.
* Added a bytecode header containing a magic number, version, and bytecode length to detect corrupted or invalid files.
* Implemented trace mode to simplify debugging and visualize execution.

---

# Limitations

* Integer arithmetic only (`i64`)
* No branching or jump instructions
* No loops
* No functions or call stack
* No labels or assembler macros
* Fixed-size global memory
